## Render Deployment Checklist

### API service

- Service name: `copmec-api`
- Runtime: `Node`
- Root directory: `backend`
- Build command: `npm install && npx prisma generate`
- Pre-deploy command: `npx prisma db push --skip-generate`
- Start command: `npm start`
- Health check path: `/api/health`

Required environment variables:

- `NODE_ENV=production`
- `NODE_VERSION=22`
- `DATABASE_URL` from Render Postgres
- `CORS_ALLOWED_ORIGINS=https://<frontend-domain>`
- `SESSION_COOKIE_NAME=copmec_session`
- `SESSION_SECRET` generated by Render
- `SESSION_TTL_SECONDS=43200`
- `SESSION_COOKIE_SAMESITE=lax`
- `PUBLIC_LOGIN_DIRECTORY_ENABLED=false`
- `MASTER_USERNAME=Maestro`
- `MASTER_PASSWORD=<secure value>`
- `JSON_BODY_LIMIT=1mb`
- `URLENCODED_BODY_LIMIT=1mb`
- `RATE_LIMIT_WINDOW_MS=900000`
- `RATE_LIMIT_MAX_REQUESTS=300`
- `RATE_LIMIT_AUTH_MAX_REQUESTS=12`
- `RATE_LIMIT_UPLOAD_MAX_REQUESTS=20`
- `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET` or `CLOUDINARY_URL`

### Web service

- Service name: `copmec-web`
- Runtime: `Static Site`
- Root directory: `frontend`
- Build command: `npm install && npm run build`
- Publish directory: `dist`

Required environment variables:

- `NODE_VERSION=22`
- `VITE_API_BASE_URL=https://<api-domain>`
- `VITE_API_URL=https://<api-domain>`

### Smoke test

1. Open `https://<api-domain>/api/health` and confirm HTTP 200.
2. Open `https://<web-domain>` and verify the login screen loads without console errors.
3. Log in with a Lead user and confirm session persistence after refresh.
4. Create a board, add a row, edit a cell, and verify the changes survive reload.
5. Import one CSV and one XLSX file and verify rows plus formulas are preserved.
6. Log in with a Junior user and verify blocked actions return `403` on unauthorized endpoints.
7. Review `/api/auth/security-events` with an admin account and confirm denied actions are recorded.
